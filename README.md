EstuFin

Es una aplicación móvil y web diseñada para que jóvenes universitarios que trabajan puedan administrar sus finanzas personales mediante el uso de "carpetas inteligentes", registros estructurados y la generación automática de planes financieros.

🎯 1. Objetivo General

Desarrollar una aplicación móvil/web orientada a jóvenes universitarios que trabajan para mejorar la administración de sus finanzas personales mediante la organización del dinero en categorías, el registro estructurado de ingresos y gastos, y la generación de planes financieros personalizados y metas de ahorro, con el fin de fomentar hábitos responsables y reducir la mala gestión económica en esta población.

🌍 2. Contexto de Uso

¿Quién?: Jóvenes de 18 a 28 años, estudiantes universitarios con empleos de medio tiempo o tiempo completo que manejan ingresos variables y poseen poca educación financiera formal.

¿Cómo?: Se utilizará principalmente en dispositivos móviles con conexión a internet para realizar registros diarios o semanales de gastos e ingresos, permitiendo separar el dinero para gastos fijos y planificar metas económicas.

📋 3. Requerimientos del Sistema
3.1 Requerimientos Funcionales


RF01: Gestión de Cuentas (Efectivo/Digital): El sistema permitirá separar el saldo entre efectivo y cuentas bancarias o tarjetas.


RF02: Registro Ágil de Gastos/Ingresos: Permitirá registrar transacciones rápidas, incluyendo micro-gastos con etiquetas personalizadas.


RF03: Módulo de Metas con Propósito: El usuario podrá crear metas de ahorro con nombre, monto objetivo y fecha límite.


RF04: Sistema de Notificaciones/Recordatorios: Enviará alertas para recordar metas de ahorro y advertir sobre el presupuesto disponible.


RF05: Visualización de Reportes: Generación de gráficos que desglosen gastos por categorías de forma sencilla.

3.2 Requerimientos No Funcionales


RNF01: Diseño Responsive (Mobile-First): Interfaz optimizada para smartphones.


RNF02: Usability (UX): Registro de gastos en menos de 3 pasos o 10 segundos.


RNF03: Seguridad de Datos: Implementación de protocolos de cifrado para proteger información financiera y credenciales.


RNF04: Disponibilidad: El servicio web debe estar operativo el 99.9% del tiempo
